## 1. Architecture Review (as-built, verified)

The 2026-07-13 audit docs (`CURRENT-STATE.md`, `PRIORITIZED-AUDIT.md`, `TARGET-STATE.md`)
describe a system where most P0 findings are **FAIL** and G1–G8 are open. The 2026-07-17
working tree has closed G1–G8 and most F-findings **in code** — but the docs were never
updated, so they are now **stale and misleading**. Worse, two *new* violations have

that is supposed to prove paper==replay is **red**. Zero-parity is unproven, not closed.

## 3. Gap Analysis (verified)

### P0 — money / safety (live)
- **GAP-1 (F2, OPEN): Zero-parity not delivered.**
  - `analytics/replay/engine.py:124` calls `get_oms_backtest_factory()(trading_context, …)`
    only when `trading_context` is passed. All production call sites
    (`interface/ui/commands/analytics_replay.py:96`, `interface/api/routers/replay.py:210,217`,
    `analytics/replay/orchestrator.py:318`, `analytics/facade.py:380,382`) construct
    `ReplayEngine(...)` with **neither** `trading_context` nor `allow_simulate_without_oms`,
    so they fall into `self._oms_adapter = None` → **PURE_SIM**. No risk/idempotency/OMS
    events fire on the live analytics path.
  - `tests/integration/quant/test_paper_replay_parity.py::test_paper_replay_parity_same_trades`
    raises `TypeError: create_oms_backtest_adapter() missing 1 required positional argument:
    'trading_context'`. The factory registry (`domain/runtime_hooks.py`) is never populated in
    the test process, so the characterization gate is **red**, not green.
  - Risk: "backtest == live path" assumption is false for the default path. Silent.
- **GAP-2 (F2a residual): slippage/commission single-source not proven.** The audit's
  `OmsBacktestAdapter` single-slippage site exists, but because the live path is SIM, the
  guarantee is unexercised. Cannot confirm paper==replay until GAP-1 fixed + parity test green.

### P1 — layering / contract
- **GAP-3 (Invariant #4): `domain/constants/ws.py` reads `os.getenv`** (`:14,17,20`) at
  import time. Domain is the most-sacred layer; this breaks the independence contract and
  fails `test_single_config`. Small, easy, must fix.
- **GAP-4 (F7/P2): `TradingContext` god-class regrowth.** `application/oms/context.py` =
  **739 LOC** (exceeds its own 525 exemption; `test_file_size_limit` red). Audit wanted it
  shrunk under Phase 4; it grew instead. Also `application/trading/trading_orchestrator.py`
  = 610 LOC. These two concentrate OMS lifecycle + PnL + shutdown + recon wiring.

### P2 — structure / duplication
- **GAP-5: `analytics/` fragmentation.** 30+ subpackages; `stocks`/`ranking`/`scoring`/`views`
  overlap (equity selection/scoring); `indicators` vs `features` overlap; `reports`/`views`/
  `visualizations` near-duplicate. Merge candidates, not yet measured for dead weight.
- **GAP-6: `domain/` over-fragmentation.** `primitives`/`models`/`value_objects`/`entities`/
  `aggregates` hold overlapping value-object concepts. `Money` canonical but re-exported from
  3 dirs. Needs consolidation, not new code.
- **GAP-7: broker god-classes.** `brokers/dhan/data/depth_feed_base.py` = **708 LOC**,
  `brokers/dhan/websocket/market_feed.py` = 637, `brokers/dhan/api/http_client.py` = 631.
  Audit undercounted these. Upstox/Dhan share `brokers/common` partially but keep separate
  `wire.py` (521 / 272 LOC). Acceptable for now (P2), flag for Phase 4.
- **GAP-8: `authenticate()` LSP non-uniform.** Dhan = token-status no-op; Upstox = reconnect;
  Paper = absent. Same `BrokerAdapter` method, three behaviors (audit F/P2-4.1).

### P3 — latent / typing
- **GAP-9: `asyncio.run` in composition root path** still possible under API event loop
  (`runtime/factory` + `event_loop`). Tracker claims moved to `Runtime.start()`; verify under
  FastAPI lifespan before trusting (audit R-async).
- **GAP-10: `Money.__eq__` coerces str/int** — masks type bugs (audit T2). Low priority.
